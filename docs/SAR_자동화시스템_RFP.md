## 1. 사업 개요

### 1.1 배경

```
┌─────────────────────────────────────────────────────────────┐
│                      현재 상황                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  SMART 원자로 수출 추진 중                                   │
│       │                                                     │
│       ▼                                                     │
│  수입국 규제기관에 SAR 제출 필요                             │
│       │                                                     │
│       ▼                                                     │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              문제점                                  │   │
│  ├─────────────────────────────────────────────────────┤   │
│  │  1. SAR 작성에 수년 소요 (19개 챕터, 수천 페이지)    │   │
│  │  2. 수입국별 규제 요건 상이 (미국, 사우디, 등)       │   │
│  │  3. 설계 변경 시 문서 전체 수정 필요                 │   │

│  │  5. NRC 질의 대응 시 근거 추적 시간 과다             │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 목적

**SMART SAR 문서의 작성/검토/관리를 자동화하여 수출 인허가 기간 단축**

| 목표 | 현재 | 목표 | 개선율 |
|------|------|------|--------|
| SAR 초안 작성 | 24개월 | 6개월 | 75% 단축 |
| 설계 변경 반영 | 2주 | 1일 | 93% 단축 |
| 규제 질의 대응 | 3일 | 2시간 | 94% 단축 |
| 문서 간 불일치 | 발견 어려움 | 자동 탐지 | - |

### 1.3 적용 범위

| 구분 | 대상 |
|------|------|
| 원자로 | SMART (System-integrated Modular Advanced ReacTor) |
| 규제 프레임워크 | US NRC (10 CFR Part 52) 우선, 추후 타국 확장 |
| 문서 유형 | SAR (Safety Analysis Report) 전 챕터 |

## 2. 현행 업무 분석

### 2.1 SAR 작성 프로세스 (As-Is)

```
┌─────────────────────────────────────────────────────────────┐
│                    현재 SAR 작성 프로세스                    │
└─────────────────────────────────────────────────────────────┘

  설계팀                    문서팀                    검토팀
    │                         │                         │
    │  설계 데이터            │                         │
    │  (도면, 계산서)         │                         │
    ├────────────────────────►│                         │
    │                         │  수작업 문서화          │
    │                         │  (Word/HWP)            │
    │                         │      │                 │
    │                         │      ▼                 │
    │                         │  SAR 초안              │
    │                         ├────────────────────────►│
    │                         │                         │  검토
    │                         │◄────────────────────────┤  코멘트
    │                         │                         │
    │                         │  수정 (반복...)         │
    │                         │      │                 │
    │                         │      ▼                 │
    │                         │  최종 SAR              │


문제점:
├── 수작업 의존 → 시간 과다, 오류 발생
├── 설계 변경 → 전체 문서 수동 검토 필요
├── 일관성 검증 → 사람 눈으로 확인 (누락 위험)
└── 근거 추적 → 파일 뒤지기 (시간 낭비)
```

### 2.2 Pain Points

| 번호 | 문제 | 영향 | 빈도 |
|:---:|------|------|:----:|
| P1 | 설계 데이터 → SAR 변환 수작업 | 작성 시간 증가 | 매일 |
| P2 | 챕터 간 수치/용어 불일치 | NRC 지적, 재작업 | 자주 |
| P3 | 요구사항 충족 여부 수동 확인 | 누락 위험 | 자주 |
| P4 | 설계 변경 시 영향 범위 파악 어려움 | 수정 누락 | 가끔 |
| P5 | NRC RAI 대응 시 근거 찾기 어려움 | 대응 지연 | 자주 |
| P6 | 수입국별 규제 차이 대응 | 재작성 필요 | 가끔 |

## 5. 프로젝트 범위 및 일정

### 5.1 단계별 범위

```
┌─────────────────────────────────────────────────────────────────────┐
│                        프로젝트 단계                                 │
└─────────────────────────────────────────────────────────────────────┘

Phase 1: 데이터 준비 (3개월)
├── 규정 문서 파싱 (SRP, DSRS, RG 1.206)
├── NuScale DCA 패턴 추출
├── 요구사항 목록 구축
├── 매핑 테이블 초안
└── 산출물: 구조화된 데이터셋

Phase 2: 지식그래프 구축 (3개월)
├── 온톨로지 설계 (DIAMOND 기반)
├── Neo4j 그래프 구축
├── 매핑 관계 적재
├── GraphRAG 파이프라인
└── 산출물: 지식그래프 + 검색 시스템

Phase 3: 에이전트 개발 (4개월)
├── Tool 개발 (검색/조회/생성/검증/추적)
├── 에이전트 워크플로우 구현
├── UI 개발
├── 통합 테스트
└── 산출물: SAR 작성 에이전트 시스템

Phase 4: 파일럿 및 고도화 (2개월)
├── Chapter 4 파일럿 적용
├── 피드백 반영
├── 성능 최적화
└── 산출물: 검증된 시스템 + 운영 가이드
```
