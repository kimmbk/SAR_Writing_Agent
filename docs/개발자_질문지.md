### Q3. VectorDB만으로 충분한가? vs 지식그래프도 필요한가?

```
Option A: VectorDB만 사용
┌─────────────────────────────────────────────────────────────┐
│  문서 → 청크 → 임베딩 → VectorDB                            │
│                    │                                        │
│                    └→ 유사도 검색으로 관련 문서 찾기         │
│                                                             │
│  장점: 구현 간단                                            │
│  단점: "관계"를 모름 (요구사항↔SAR↔설계 연결 불가)          │
└─────────────────────────────────────────────────────────────┘

Option B: VectorDB + 지식그래프
┌─────────────────────────────────────────────────────────────┐
│  문서 → 파싱 → 엔티티/관계 추출 → Neo4j (지식그래프)        │
│              → 청크 → 임베딩 → VectorDB                     │
│                                                             │
│  장점: 관계 추적 가능 (추적성, 영향분석)                    │
│  단점: 구현 복잡, 매핑 작업 필요                            │
└─────────────────────────────────────────────────────────────┘

질문:
1. 우리 목표(추적성, 일관성 검증, 영향분석) 생각하면
   지식그래프 필수인 거 맞지?
2. 아니면 VectorDB만으로도 가능한 방법이 있나?
3. 둘 다 쓴다면 역할 분담은 어떻게?
   - VectorDB: 의미 기반 검색 (유사 문장 찾기)
   - 지식그래프: 관계 기반 검색 (요구사항→SAR 추적)
   이렇게?
```
